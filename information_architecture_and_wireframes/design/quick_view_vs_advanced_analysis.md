# Quick View vs Advanced Analysis

## Default path: Quick View
Audience:
- general public
- leadership
- communications
- first-time users

What it shows:
- plain-language purpose
- map
- top summary cards
- simplified filters
- latest readings with interpretation
- threshold colors
- event markers
- short glossary links

What it hides:
- dense parameter lists
- sensor-specific toggles
- heavy graph controls
- technical configuration details

## Secondary path: Advanced Analysis
Audience:
- researchers
- scientific staff
- data analysts

What it adds:
- parameter selection
- aggregation controls
- time-range controls
- downloadable graphs/tables
- sensor-level toggles
- raw metadata
- advanced filtering

## UX rule
Quick View should be the default.
Advanced Analysis should be entered through:
- a tab
- a toggle
- or a drill-down action

## Labeling suggestion
Use top-level toggle labels:
- Quick View
- Advanced Analysis
