# Information Architecture — Replacement Dashboard

## IA goal
Create a dashboard that is immediately understandable for a first-time visitor, while preserving scientific depth for researchers.

## Audience model
1. General public / non-specialist
2. Researcher / scientific user
3. Internal technical / data team

## Primary navigation structure
1. Landing / Map
2. Station Detail
3. Live Data
4. Metadata
5. Help / Glossary

## Page purposes

### 1. Landing / Map
Purpose:
- Explain what the dashboard is
- Show the monitoring network
- Let users find a station quickly
- Offer two pathways: Quick View and Advanced Analysis

Above the fold:
- purpose statement
- quick summary cards
- search
- main filters
- map

Below:
- recent highlights
- selected station preview
- help / glossary entry point

### 2. Station Detail
Purpose:
- Show station summary before dense graphs

Order:
1. Station title
2. Status
3. Last reading / freshness
4. Location and device type
5. Quick interpretation summary
6. Drill-down graph controls
7. Downloads / metadata links

### 3. Live Data
Purpose:
- Make latest readings easy to scan quickly

Order:
1. Station summary strip
2. current values
3. min / max / trend
4. interpretation / thresholds
5. advanced expansion area

### 4. Metadata Modal / Panel
Purpose:
- Explain variables and instrumentation without crowding the main page

Must include:
- station summary
- data availability
- instruments
- variables
- units and definitions

### 5. Help / Glossary
Purpose:
- Support first-time users

Must include:
- PM2.5
- PM10
- pressure
- humidity
- air temperature
- station types
- interpretation notes

## Design rules
- summary before detail
- interpretation before dense controls
- help always accessible
- advanced analysis must be reachable but not dominant
