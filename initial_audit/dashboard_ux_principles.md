# Dashboard UX Principles

## Core direction
The replacement dashboard must be:
- intuitive for laymen
- deep enough for scientists
- visually polished
- faster to navigate
- easier to interpret

## Design principles
1. Layman-first, scientist-expandable
   - default view should explain what the user is looking at
   - deeper analysis tools should be available without cluttering the main screen

2. Purpose-first
   - every main page should answer:
     - What am I looking at?
     - Why does it matter?
     - What can I do next?

3. Interpretation, not just raw numbers
   - users should see context, normal ranges, or plain-language meaning where possible

4. Reduced clutter
   - less-used controls should move into tabs, accordions, or drill-down areas

5. Strong visual hierarchy
   - summary at the top
   - detail below
   - clearer status treatments
   - cleaner card and filter structure

6. Mobile and touch awareness
   - map and panels must resize well
   - interactions must remain usable on smaller screens
